# Spring Boot 3.5.8 Improvements Over 3.5.7

## Snyk SCA Status

**Both versions show identical SCA results:**
- Dependencies: 105 (same)
- Vulnerabilities: 1 Low Severity (same)
- Medium/High/Critical: 0 (same)

## Dependency Version Improvements

### Spring Boot 3.5.7
- Tomcat: 10.1.48
- Spring Framework: 6.2.12
- Netty: 4.1.128.Final

### Spring Boot 3.5.8
- Tomcat: 10.1.49 (newer)
- Spring Framework: 6.2.14 (newer)
- Netty: 4.1.128.Final (same)

## Why 3.5.8 is Better

Even though Snyk shows the same vulnerability count, Spring Boot 3.5.8 includes:

1. **Newer Tomcat** (10.1.49 vs 10.1.48)
   - Latest bug fixes and improvements
   - Better performance and stability

2. **Newer Spring Framework** (6.2.14 vs 6.2.12)
   - Latest bug fixes
   - Performance improvements
   - Additional features and enhancements

3. **57 Bug Fixes** (per Spring Boot release notes)
   - Various improvements and fixes
   - Better stability and reliability

4. **Future-Proofing**
   - More recent versions
   - Better support and maintenance
   - Aligned with latest Spring ecosystem

## Conclusion

While Snyk SCA shows the same vulnerability count, Spring Boot 3.5.8 is better because:
- ✅ Newer dependency versions with latest fixes
- ✅ 57 bug fixes and improvements
- ✅ Better performance and stability
- ✅ More future-proof

The single low-severity vulnerability (Kotlin stdlib via okhttp) exists in both versions and is unrelated to Spring Boot itself.
